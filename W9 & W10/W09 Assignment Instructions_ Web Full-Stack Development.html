    W09 Assignment Instructions

    In this assignment, we will use Angular pipes to give end users the ability to search and filter the list of contacts by name. We will also learn how to make HTTP requests to get and save the data permanently to a back-end database server.

    Angular Pipes

    Theoretically, there could be many, many contacts displayed in the contact list in the cms application. It would be nice to add a feature where the end user could search for contacts by name, instead of having to scroll through a very long list. The end user would enter a search value in the ContactListComponent, and then the application would automatically filter out only those contacts whose name contained the value that they entered. This can easily be done by using an Angular pipe.

    Angular pipes are analogous to using operating system pipes where the output of one command is piped into the input of another command. In the example below, the output of the Bash shell ls command is sent as an input to the more command:

    ls | more

    Angular pipes allow us to transform data from one format to another. Output data from some source is sent into the pipe where it is transformed. The transformed data coming out of the pipe and is sent to the target location.

    You will need to first create an Angular pipe to filter out the contacts and then modify the ContactListComponent to give the end user the ability to search for contacts by name.

    Create an Angular Pipe to Filter Out Contacts

    A custom Angular pipe is needed to filter the list of contacts by name. A pipe is a special type of object in Angular and is implemented in its own class. Inside the class, you must create and implement the transform() method. This method is responsible for transforming the data sent into the pipe into a new format.

    Follow the directions below to create a new pipe to filter the list of contacts by name:

    1. Open the terminal in VS Code. Change to the contacts folder and use the ng command to create a new pipe named contacts-filter in the same folder:

      ng g p contacts-filter --flat

    2. Open the contacts-filter.pipe.ts file. Notice that the generated file implements the PipeTransform interface and created the following method for you:

      transform(value: any, args? : any): any

      The value parameter contains the data input to the pipe that is to be transferred into a diﬀerent format. The args parameter contains an array of one or more values that are needed to transform the data. When the ContactsFilterPipe is used in the cms application, the value parameter will contain the array of contacts in the contact list. The args parameter will contain the term or search value entered by the end user. The transform method returns the transformed data. It will return an array of only those Contact objects whose name contains the term (search value) entered by the end user. Modify the method signature of the transform method to use more meaningful names for the two input parameters:

      transform(contacts: Contact[], term: string): any

    3. Here is the algorithm for the transform method to filter out all of the contacts whose name contains the value of the search term:

      transform(contacts: Contact[], term) BEGIN
         
         Create a new array to contain the filtered list of contacts
       
         FOR every contact in the contacts list
            IF the contact's name includes the value of the search term THEN
               Add the contact to the new filtered array
            ENDIF
         ENDFOR
       
         If the new filtered array has no contacts in it THEN
            RETURN the original contacts list
         ENDIF
       
         RETURN the new filtered array of contacts
      END

      Here is one implementation of the transform function:

      transform(contacts: Contact[], term) { 
          let filteredArray: Contact[] =[];  
          for (let i=0; i<contacts.length; i++) {
             let contact = contacts[i];
             if (contact.name.toLowerCase().includes(term)) {
                filteredArray.push(contact);
             }
          }
          if (filteredArray.length < 1){
             return contacts;
          }
          return filteredArray;
         }

      We can simplify this implementation by using the JavaScript filter() array method to replace the for loop. The filter() method will accomplish the exact same thing as the for loop above.

      A fat arrow function must be passed to the filter() method. The fat arrow function is called for every element in the array. It must contain a condition to determine if the current element passes the condition. If the fat arrow function returns true, the element is added to a new filtered array. The filter() method returns the new filtered array after it has gone through all of the elements in the array.

      Here is the implementation of the transform() method using the array filter() method:

      transform(contacts: Contact[], term) { 
         let filteredContacts: Contact[] =[];  
         if (term && term.length > 0) {
            filteredContacts = contacts.filter(
               (contact:Contact) => contact.name.toLowerCase().includes(term.toLowerCase())
            );
         }
         if (filteredContacts.length < 1){
            return contacts;
         }
         return filteredContacts;
      }

      The fat arrow function passed to the filter() array method above receives a reference to the next Contact object in the contacts array. The fat arrow function gets the value of the name property of the Contact object passed to the function, and then converts it to lowercase letters using the JavaScript toLowerCase() String method. It then calls the JavaScript includes() String method. The search term passed as an input into the transform() method is converted into lowercase letters and passed to the includes() method. The includes() method returns true if the contact’s name contains the value of the term, and the Contact object is then added to the filtered array.

    4. Save the contacts-filter.pipe.ts file.

    Making an HTTP Request in Angular

    Angular gives us a very easy way to submit HTTP requests to get, insert, update, and delete (C.R.U.D.) data from a web or application server. Most application servers today implement this functionality as RESTful web services.

    RESTful web services take advantage of the existing HTTP protocol. The HTTP protocol defines different types of requests. Five of these include GET, POST, PUT, PATCH, and DELETE. RESTful web services associate a meaning with each of these types of HTTP requests as shown below:

    HTTP

    Request Type

    RESTful Meaning

    Return Code

    GET

    Read/Query to get one or more objects

    200 (Success), 404 (Not found)

    POST

    Create/Insert a new object

    201 (Success), 404 (Not found), 409 (Object already exist)

    PUT

    Replace/Update an existing object

    200 (Success), 204 (No content), 404 (Object being replaced was not found)

    PATCH

    Modify/Update an existing object

    200 (Success), 204 (No content), 404 (Object being replaced was not found)

    DELETE

    Delete an existing object

    200 (Success), 404 (Object not found)

    In this part of the assignment, we will use Angular to make HTTP GET requests to get all of the Documents, Messages, and Contacts stored in a database, and make HTTP PUT requests to update changes made to them. The database will be stored in Google’s Firebase database server.

    Making HTTP Get and Put Requests

    Now we need to add methods to the DocumentService, MessageService, and ContactService to issue the HTTP Get and Put requests to retrieve and update these collections in the database. These methods will be called whenever you need to Create, Read, Update, and Delete (C.R.U.D.) documents in your Firebase database.

    Use the getRecipes() and storeRecipes() methods in the data-storage.service.ts file in the Recipe Book project for an example of how these methods should be implemented. NOTE: You need to be aware of one major diﬀerence: the getRecipes() and storeRecipes() methods are using an older, deprecated HttpModule. This module required that you called the map() method to transform the data returned from the Firebase server from a string format to a JSON object. The newer HttpClientModule does the mapping for you automatically, so you do not need to call the map() method. The instructions below use the newer HttpClientModule.

    Modify the DocumentService to Issue an HTTP Get Request

    1. Open the document.service.ts file and replace the code in the getDocuments() method.
      1. Import the HttpClient and HttpHeaders packages at the top of the file.
        import { HttpClient, HttpHeaders } from '@angular/common/http';
      2. Inject the HttpClient object into the DocumentService class through dependency injection.
      3. Call the HTTP service’s get() method to make an HTTP Get request to get the array of documents from your Firebase database server. It returns an Observable object because all HTTP requests are asynchronous (i.e. the response will not be returned immediately). This Observerable object waits and listens for a response to be returned from the server.
      4. You then need to call the Observable class’s subscribe() method to create a subscription to watch for the response returned from the database server. You must pass the subscribe() method two inputs: a success method and an error method. The success method is executed when the HTTP Get request is successful. The error method is executed when an error is returned from the server. Below are the algorithms for the success and error methods:
        .subscribe(
           // success method
           (documents: Document[] ) => {
              this.documents = documents
              this.maxDocumentId = getMaxId()
              sort the list of documents
              emit the next document list change event
           }
           // error method
           (error: any) => {
              print the error to the console
           } 
        }
      5. The success method is implemented as an anonymous, fat arrow function. It is executed when the HTTP Get request is successful. This function has one input, an array of documents passed into the function when a response is received from the database server. The function should do the following:
        1. Assign the array of documents received to the documents property.
        2. Call the getMaxId() method to get the maximum value used for the id property in the document list, and assign the value returned to the maxDocumentId property.
        3. Sort the list of documents by name using the sort() JavaScript array method. This method requires that you pass it a compare function. The compare function is called for each element in the array. It receives two inputs, the current element in the array and the next element in the array. If the current element is less than the next element, return a minus one. If the first element is greater than the next element, return a one; else, return zero.
        4. Emit the documentListChangedEvent subject and pass it a cloned copy of the documents array to notify the DocumentListComponent that the document list has changed. Use the slice() JavaScript array method to get a clone of the documents array.

    2. You should always assign an initial value to arrays and strings whenever you define a new variable. Make sure that you have initialized the documents property with an empty array as shown below:

      documents: Document[] = [];

      If you do not do this you will get an error when the DocumentListComponent first loads because it will call the DocumentService getDocuments() method to get the list of documents. The HTTP Get request may not have returned the data back from the server yet and the list would have an undefined value and result in an error.

    3. Save all of your changes, start your application, and view your application in the browser. The list of documents should appear as before, except that the list of documents is being retrieved from the Firebase database server.

    Modify the DocumentService to Issue an HTTP Put Request

    1. Create the storeDocuments() method in the DocumentService class. This method will be called when a Document object is added, updated, or deleted in the document list. It will issue an HTTP Put request to update the document list in your Firebase database server. This method should do the following:

      1. Convert the documents array into a string format by calling the JSON.stringify() method. This is important because only text data can flow through firewalls when an HTTP request is made.
      2. Create a new HttpHeaders object that sets the Content-Type of the HTTP request to application/json.
      3. Call the http service’s put() method to send the document list to your Firebase database server.
      4. The put() method returns an Observable object because HTTP requests are asynchronous (for example, the response will not be returned immediately). Call the Observable subscribe() method to perform the desired action when the server returns with a successful response. Pass a fat arrow function that will be executed when the response is returned from the database server. There are no inputs passed to the subscribe() method. This function only needs to emit the documentListChangedEvent and pass it a cloned copy of the documents array to notify the DocumentListComponent that the document list has changed.

    2. Modify the addDocument(), deleteDocument() and updateDocument() methods. Delete and replace the last statement (the call to the documentListChangedEvent's next() method) with a statement to call the storeDocuments() method instead.

    3. Save all of your changes, open your browser, and select the Documents feature. The list of documents should now be displayed as before. Click on one of the documents in the list, edit it, and save it. Select another document in the list and delete it. The changes should have been saved in your Firebase database server. Refresh your browser and view the document that you edited. The document that you deleted should be deleted from the document list. This proves that the changes you made were successfully saved to the database server.

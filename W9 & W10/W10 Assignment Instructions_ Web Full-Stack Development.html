    W10 Assignment Instructions

    In this assignment, you will begin developing the back-end web services for the CMS application. These back-end web services typically run on a server located somewhere in the network.

    There are a variety of different technologies that allow you to build back-end web services such as Java, .NET, and PHP. These solutions are all written in programming languages other than JavaScript. This requires two different sets of skills to develop a web-based application. These technologies were designed to dynamically build the web pages on the back end. This approach limits the type of dynamic interactive experience end users have come to expect because many requests and more data need to be transported over the network. These solutions can also be very expensive to deploy and maintain. They also typically use relational databases that require the joining of many tables together to obtain the data required to display a web page.

    Angular, NodeJS, and NoSQL databases provide an alternative architecture that strives to overcome these limitations. JavaScript is used to develop both the front end and back end of your application. This allows developers to work in one programming language. NodeJS is a very light-weight server that scales extremely well to large numbers of users and is designed to be deployed easily to a cloud environment. No special software and fewer servers are required to support a NodeJS application. NoSQL databases, such as MongoDB, are designed to store data in a document structure so that few joins are required to get the data needed for a web page.

    You will use NodeJS with the Express package, and eventually, MongoDB, to develop RESTful web services to create, read, update and delete the documents, messages, and contacts stored in a MongoDB database for the CMS application.

    Creating a NodeJS/Express Project

    You have already installed and have been using NodeJS and npm from the beginning of Week 01. You've used the ng command-line interface (CLI) to create an Angular project and various components, services, and directives. Each time you typed ng serve to run your program, you were actually starting up a NodeJS application server. The NodeJS server is designed to help you build and test your Angular applications.

    You will also use the NodeJS package to build a back-end server to implement the basic web services to create, read, update and delete (CRUD) the documents, messages, and contacts that will be stored in a MongoDB database in the next lesson.

    NodeJS by itself is a very simple application server with little functionality. The Express add-on package provides additional functionality that turns NodeJS into a web server that implements user sessions and server-side routing to allow you to serve up both static and dynamic content.

    Follow the instructions below to create a NodeJS web server using Express:

    1. Download and install the Express packages in the CMS project. NodeJS provides a wonderful command called npm (Node Package Manager) that allows us to download, install, and uninstall NodeJS packages that are needed for your project. Open the terminal window in VS Code. Type the following command to install the required packages for Express:

      npm install --save express body-parser cookie-parser morgan

    2. Create a new folder inside of the CMS folder named server. This is where all of the files needed for your back-end web services will be located. Create two additional folders named models and routes inside of the server folder.



    3. Create a new file in the CMS folder and save it with the name of server.js. This is the file that will start our NodeJS server.

    4. Download and extract the Lesson 10 Files download . Open the lesson10 folder in a new instance of VS Code. Open the server.js file and copy the contents of the file into the server.js file of your project. Save the file.

      Notice that the contents of this file are written in JavaScript and not in TypeScript. The server.js file creates and configures an instance of Express and creates an HTTP server that listens for HTTP requests on a specified port.

      Notice the use of the NodeJS require() function at the top of the file. The require statement in JavaScript is similar to the import statement in TypeScript. It is used to tell NodeJS what other files or packages are required and referenced in this file.



    5. Study the rest of the comments in the server.js file to get a better understanding of what each statement is doing.
